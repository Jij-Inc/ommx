# Release Notes

Release notes for the `ommx` crate are organized by minor release line.

Each line page starts with the design or migration context for that line, then
keeps patch, alpha, beta, and release-candidate updates in a short update
section. New minor lines should get their own page, such as `3.1.md`, instead
of extending the previous line indefinitely.
