# OMMX Python SDK 3.0.x

## 3.0.0 Alpha 1

```{note}
This is a pre-release version. APIs may change before the final 3.0.0 release.
```

The theme of this release is the **complete migration from Python/Protocol Buffers to Rust/PyO3**. In 2.0.0, the core implementation was rewritten in Rust while Python wrapper classes remained for compatibility. In 3.0.0, those Python wrappers are removed entirely — all `ommx.v1` types are now direct re-exports from Rust, and the `protobuf` Python runtime dependency is eliminated.

A migration guide for use with [Claude Code](https://www.anthropic.com/claude-code) and other AI assistants is available in the [Python SDK v2 to v3 Migration Guide](https://github.com/Jij-Inc/ommx/blob/main/PYTHON_SDK_MIGRATION_GUIDE.md).

### Complete Rust re-export of `ommx.v1` types ([#770](https://github.com/Jij-Inc/ommx/pull/770), [#771](https://github.com/Jij-Inc/ommx/pull/771), [#774](https://github.com/Jij-Inc/ommx/pull/774), [#775](https://github.com/Jij-Inc/ommx/pull/775))

All `ommx.v1` classes — `Linear`, `Quadratic`, `Polynomial`, `Function`, `DecisionVariable`, `Parameter`, `Constraint`, `RemovedConstraint`, `NamedFunction`, `Instance`, `ParametricInstance`, `Solution`, `SampleSet` — are now implemented entirely in Rust with no Python wrapper layer. This removes over 4,500 lines of Python code and eliminates the `.raw` attribute that provided access to the underlying protobuf objects.

Operators, constraint ID management, DataFrame properties, QUBO/HUBO pipelines, relaxation, and sampling helpers are all part of the native implementation. The `State` type accepts `dict`, `Mapping`, or `Iterable` from Python.

### Removal of Protocol Buffers Python dependency ([#776](https://github.com/Jij-Inc/ommx/pull/776))

All 28 `*_pb2.py` / `*_pb2.pyi` files have been deleted and the `protobuf` runtime dependency is removed. All Python types are now served exclusively by native bindings; serialization to/from the protobuf wire format is handled internally.

### Annotation system reimplemented ([#772](https://github.com/Jij-Inc/ommx/pull/772))

The `annotations` field on `Instance`, `ParametricInstance`, `Solution`, and `SampleSet` is now implemented natively. The Python-side `UserAnnotationBase` class and `annotation.py` descriptors are removed.

### Artifact/ArtifactBuilder reimplemented ([#782](https://github.com/Jij-Inc/ommx/pull/782))

`Artifact` and `ArtifactBuilder` (including Archive/Dir/Builder variants) are reimplemented natively with `ommx.artifact` module auto-generated. The six Python-only subclasses are removed and unified.

### Auto-generated `ommx/v1/__init__.py` ([#779](https://github.com/Jij-Inc/ommx/pull/779))

The hand-written 361-line `v1/__init__.py` is now auto-generated, produced by `task python:stubgen` via the `generate-init-py` feature of `pyo3-stub-gen`.

### Consolidated pandas DataFrame logic ([#778](https://github.com/Jij-Inc/ommx/pull/778))

The internal DataFrame conversion logic is consolidated and null handling is unified to `pandas.NA`.

### API reference switched to pyo3-stub-gen docgen ([#780](https://github.com/Jij-Inc/ommx/pull/780))

The API documentation for `ommx.v1` types is now generated by `pyo3-stub-gen`'s `doc-gen` feature instead of `sphinx-autoapi`. Pure-Python adapter packages continue to use `sphinx-autoapi`.

### Documentation migrated to Sphinx ([#785](https://github.com/Jij-Inc/ommx/pull/785))

Documentation has been migrated from Jupyter Book (GitHub Pages) to Sphinx with `myst-nb` (ReadTheDocs). The theme is switched to `sphinx-book-theme` with KaTeX for math rendering. English and Japanese docs are hosted as separate RTD projects with shared API reference.
